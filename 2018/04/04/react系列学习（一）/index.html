react系列学习（一）

2018-04-04 react 评论

目前，开始系统学习react。计划在学习过程中，记录学习过程中的问题和经验，以此巩固知识。本文记录React入门教程的学习过程。该教程从一个游戏开始，循序渐进的讲解了react的核心部分，本文只是个人的学习记录和总结。

如何开始本地开发

在开始实践一门新的语言或框架之前，我们需要完成的第一步就是搭建本地开发环境。教程请点击如何创建并开始一个react app

react的单向数据流

子元素通过props获取父元素传递的数据

在父元素引用子元素时，子元素采用类似html元素属性的形式获取参数，如下：

<Board
             squares = {current.squares}
             onClick = {(i) => this.handleClick(i)}
             winnerLine = {winnerLine}
           />

在子元素内部，通过this.props获得从父元素传递来的参数。

<Square key={i}
              value={this.props.squares[i]}
              onClick={() => this.props.onClick(i)}
              highlight={this.props.winnerLine.includes(i)}
      />

子元素修改父元素的数据

按照react单向数据流的设计模式，我们不能直接在子元素内部修改父元素的数据。

惯例的做法是：通过props传递一个父组件当中的事件处理函数到子组件中。

组件的构造函数

在constructor()中，必须调用super()，方法才能在继承父类的子类中正确获取到类型（疑问：什么叫类型的this）的this。

状态提升

当遇到同时获取多个子组件数据，或两个子组件之间需要相互通讯的情况时，把子组件的state数据提升至其共同的父组件当中保存。之后父组件可以通过props将状态数据传递到子组件中，这样应用当中的状态数据就能够更方便地交流共享了。

<!-- Square组件 -->
class Square extends React.Component {
  render() {
    return (
      <button className="square" onClick={() => this.props.onClick()}>
        {this.props.value}
      </button>
    );
  }
}
<!-- Boared中调用子组件 -->
 return (
      <Square
        value={this.state.squares[i]}
        onClick={() => this.handleClick(i)}
      />
    );

在上面的代码中，每次被点击都会触发传人的onClick方法。捋一下这中间发生了什么：

  1. 添加onClick属性到内置的DOM元素button上让react开启了对点击事件的监听；
  2. 当按钮被点击时，react会调用Square组件的render()方法中的onClick事件处理函数；
  3. 事件处理函数触发了传入其中的this.props.onClick方法，这个方法是由Board传递给Square的。
  4. Board传递了onClick={() => this.handleClick(i)给Square，所以当Square中的事件处理函数被触发时，其实就是触发了Board当中的this.handleClick(i)方法。

react的不可变性

改变应用数据的方式一般分为两种。第一种时直接修改已有从变量的值；第二种则是将已有的变量替换为一个新的变量。

直接修改数据

var player = {score: 1, name: 'Jeff'};
player.score = 2;
// Now player is {score: 2, name: 'Jeff'}

替换修改数据

var player = {score: 1, name: 'Jeff'};

var newPlayer = Object.assign({}, player, {score: 2});
// Now player is unchanged, but newPlayer is {score: 2, name: 'Jeff'}

两种方式的结果是一样的，但是第二种并没有改变之前已有的数据。通过这样的方式，我们可以得到几点好处：

  1. 很轻松地实现 撤销/重做以及时间旅行
  2. 记录变化
  3. 在React当中判定何时重新渲染（如何定量分析这种优化？）

函数定义组件

对于不需要this.state的组件，我们可以去掉组件的构造函数。更进一步的，React专门为只有render方法的组件提供了一种更简便的定义组件的方法：函数定义组件。

只需要简单写一个以props为参数的function返回JSX元素就搞定了。

function Square(props) {
  return (
    <button className="square" onClick={props.onClick}>
      {props.value}
    </button>
  );
}

仔细看上面的代码，会发现把onClick={() => props.onClick}修改为onClick={props.onClick}。注意不要写成onClick={props.onClick()}。否则props.onClick方法会在Square组件渲染时被直接触发而不是等到被点击触发，会导致render()方法被循环调用。

Keys

当你在react当中渲染列表时，react都会试着存储对应每个单独项的相关信息。如果你的组件包含state状态数据，那么这些状态数据必须被拍下，不管你的组件是怎么编写实现的。

key是react中使用的一种特殊的属性。当元素被创建时，react会将元素的key值和对应元素绑定存储起来。当一个列表被重新渲染时，react会根据较新的元素依据相应的key值来匹配之前的元素内容。

当一个新的key至添加到列表当中时，表示有一个组件被创建；被删除时表示有一个组件被销毁；改变某个组件的key值，它会在下次渲染时被销毁并当作新的组件重新渲染。

强烈建议在渲染列表时添加keys值。
