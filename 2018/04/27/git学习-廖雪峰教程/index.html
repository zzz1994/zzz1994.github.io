git系列学习（一）-廖雪峰教程

2018-04-27 评论

最近准备系统学习git，会学习几个相关教程。在学习过程中，会记录学习中的重点和问题，以此巩固，并方便后续的查询。

本地创建版本库

  1. 新建一个空目录，并打开：

    mkdir git-study
cd git-study

  2. 在目录中创建一个版本：

    git init

  3. 新增一个readme.md文件，并把文件添加到暂存区：

    git add readme.md

  4. 把文件推送到本地版本库

    git commit -m "新增readme.md"

经过上面四个步骤，就在本地建立了一个版本库。

本地版本控制

在进行各种操作之前，我们可能需要查看当前git库的各种状态。

  1. 查看当前git库的状态

    git status

  2. 如果发现git库中的readme.md有变化，我们想要查看变化内容：

    git diff readme.md

版本回退

通过上面的学习，我已经能创建一个git版本库，并不断新增/修改。现在，我在本地工作目录中新增并修改了一些文件，并添加到了版本库中，下班前，我发现这次的修改都是不需要的。

  1. 我首先查看我的提交历史

  1. 在git中，使用HEAD表示当前版本，上一个版本是HEAD^，上上个版本是HEAD^^，以此类推。一种简写方法是：HEAD~100，表示往上100个版本。

工作区和暂存区

git和其他版本控制系统如svn的一个不同之处就是有暂存区的概念。

  • 工作区：在电脑里能看到的目录。
  • 版本库：工作区有一个隐藏目录.git，这个不是工作区，而是git的版本库。

查看工作区和版本库里面最新版本的区别：

git diff HEAD -- readme.md

撤销修改

凌晨两点，在readme.md中写了一句错误代码。准备提交的时候，发现了这个错误。这时，有以下几种解决方案：

  1. 手动删除（不建议）；
  2. 使用git命令行:
git checkout -- readme.md

该命令撤销的是工作区的修改，撤销暂存区需要使用另一个命令。
